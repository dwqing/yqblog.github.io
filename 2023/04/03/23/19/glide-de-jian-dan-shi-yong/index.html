Glide的简单使用

1添加依赖

在build.gradle文件中添加以下依赖：

dependencies {    implementation 'com.github.bumptech.glide:glide:4.12.0'   
                  annotationProcessor 'com.github.bumptech.glide:compiler:4.12.0'
}

2添加网络权限

在 AndroidManifest.xml 添加网络权限：

<uses-permission android:name="android.permission.INTERNET" />

3基础的使用

1添加网络图片(项目中经常用到的)

Glide.with(context)//context是当前的activity
     .asBitMap()   //将图片转化为bitMao格式可能会比较清晰
     .load("https://www.example.com/image.jpg") //图片的网络地址
     .into(imageView); //你想要放置的图片位置


Glide.with(context)   //context是当前的activity
     .load("https://www.example.com/image.jpg") //图片的网络地址
     .into(imageView); //你想要放置的图片位置

2加载本地的图片

Glide.with(context)
     .load(new File("/path/to/image.jpg"))
     .into(imageView);

3加载资源目录图片

Glide.with(context)
     .load(R.drawable.image)
     .into(imageView);

4从Assets目录中加载图片(目前还未了解)

Glide.with(context)
     .load("file:///android_asset/image.jpg")
     .into(imageView);

4显示占位符（加载过程的图片）和错误图片

1占位符

Glide.with(context)
     .load("https://www.example.com/image.jpg")
     .placeholder(R.drawable.placeholder) // in there
     .into(imageView);

2错误情况下的图片

Glide.with(context)
     .load("https://www.example.com/image.jpg") 
     .error(R.drawable.error)  //in there
     .into(imageView);

5其他的一些格式

1缩略图（可以在加载图片时先显示缩略图，待图片加载完毕后再显示原图）

Glide.with(context)
     .load("https://www.example.com/image.jpg")
     .thumbnail(0.5f)
     .into(imageView);

2圆角图

Glide.with(context)
     .load("https://www.example.com/image.jpg")
     .transform(new RoundedCorners(30))
     .into(imageView);

3圆形图片

Glide.with(context)
     .load("https://www.example.com/image.jpg")     
     .apply(RequestOptions.circleCropTransform())
     .into(imageView);

4模糊图片

Glide.with(context)
     .load("https://www.example.com/image.jpg")
     .apply(RequestOptions.bitmapTransform(new BlurTransformation(25)))
     .into(imageView);

7杂叙

1圆形的显示

try {
               Glide.with(Main_FriendsFragment.this)
                       .load("http://"+mFriends.getAvatar())
                       .into(new CustomTarget<Drawable>() {   //这里是和网络请求一样的东西
                           @Override
                           public void onResourceReady(@NonNull Drawable resource, @Nullable Transition<? super Drawable> transition) {
                               Bitmap bitmap = ((BitmapDrawable)resource).getBitmap();
                               RoundedBitmapDrawable roundImg1=    
                               RoundedBitmapDrawableFactory.create(getResources(),bitmap);
                               roundImg1.setAntiAlias(true);
                               roundImg1.setCornerRadius(bitmap.getWidth()/2);
                               headImage.setImageDrawable(roundImg1);//设置圆形图片的显示
                           }

                           @Override
                           public void onLoadCleared(@Nullable Drawable placeholder) {

                           }
                        });

           } catch (Exception e) {
               e.printStackTrace();
           }

其解释：

Glide.into(new CustomTarget())是使用Glide加载图片到一个自定义的目标中。

其中，CustomTarget表示自定义的目标类型，它是Glide提供的一个接口，用于自定义图片的加载和显示方式。这个接口有两个方法：

  • onResourceReady(T resource, Transition<? super T> transition)：当图片加载成功时，Glide会回调这个方法，并将加载成功的图片资源和一个过渡对象传入。
  • onLoadCleared(Drawable placeholder)：当图片加载被取消或清除时，Glide会回调这个方法，并将一个占位符Drawable对象传入。

通过调用Glide.into()方法，将自定义的目标传入，Glide会根据目标类型，调用相应的方法，完成图片的加载和显示。在这个例子中，目标类型是CustomTarget，表示加载的图片资源是一个Drawable对象。
